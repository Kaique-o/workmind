# Workmind: Master Instructions & Ideology

Este arquivo serve como a "Memória Central" do projeto Workmind para garantir consistência em todas as iterações.

## 1. O Coração do Produto
A Workmind não é apenas uma ferramenta de coleta; é uma plataforma de **Inteligência Psicossocial e Mudança de Comportamento Organizacional**.

### O Ciclo de Valor (Narrativa Master)
`Dados Anônimos` → `Análise Agregada` → `Risco Identificado` → `Insight IA` → `Plano de Intervenção` → `Feedback Loop` → `Impacto Mensurado`

---

## 2. Pilares Estratégicos (Leis do Projeto)

### Pilar 1: Confiança Radical
*   **Anonimato como Produto:** O selo de anonimato deve estar presente em todas as telas sensíveis.
*   **Pacto de Privacidade:** Todo colaborador deve passar pelo onboarding de privacidade antes de responder.
*   **Transparência:** Mostrar sempre o que a empresa *vê* vs o que ela *não vê*.

### Pilar 2: Inteligência Executiva (Empresa)
*   **IA Explicável:** Todo insight deve seguir a estrutura: **Sinal** (o que aconteceu) → **Causa Provável** (por que) → **Ação Recomendada** (como agir) → **Impacto Esperado** (resultado).
*   **Compliance NR 01:** O sistema deve gerar evidências reais para o PGR (Programa de Gerenciamento de Riscos).
*   **ROI Financeiro:** O valor deve ser provado em R$ (redução de turnover e absenteísmo).

### Pilar 3: Mobile-First Verdadeiro (Colaborador)
*   **Ergonomia de App:** Navegação inferior (Bottom Nav), botões de toque amplos (min 48px).
*   **Check-in < 60s:** Interações rápidas para evitar fadiga de pesquisa.
*   **Feedback Loop:** O colaborador precisa saber que a empresa agiu ("A Empresa Agiu").

---

## 3. Diretrizes Técnicas (Stack & UI)

*   **Tecnologia:** Apenas HTML5, CSS3 (Vanilla) e JavaScript (Vanilla).
*   **Design System:**
    *   `risk-low`: #10b981
    *   `risk-medium`: #f59e0b
    *   `risk-high`: #ef4444
    *   `risk-critical`: #b91c1c
    *   `risk-anonymized`: #6b7280
*   **Nomenclatura:** Snake_case ou camelCase consistente.
*   **Responsividade:** Mobile-First para Colaborador, Desktop-First (Executive Dashboard) para Empresa.

---

## 4. Viéses Cognitivos a Monitorar
*   **Viés de Vigilância:** Mitigar com selos de anonimato.
*   **Viés de Reciprocidade:** Gerar valor ao colaborador (insights pessoais) para que ele responda melhor.
*   **Efeito Progresso:** Usar barras de progresso e checklists (NR 01).

---

## 5. Checklist de Verificação Profissional
1. [ ] O colaborador se sente seguro para responder? (Onboarding + Selos)
2. [ ] O gestor sabe exatamente qual ação tomar? (IA Explicável)
3. [ ] A NR 01 está coberta por evidências? (Compliance Area)
4. [ ] O valor financeiro está claro para a diretoria? (ROI Dashboard)
5. [ ] A interface parece um SaaS Enterprise e não um template? (Custom UI)
